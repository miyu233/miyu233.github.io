0 小技巧

通过Header，Tooltip和Space 属性来组织Inspector中的属性显示。

0.1 Header

[Header("This is a example")]

十分常用，可以给属性添加标题

0.2 Tooltip

[Tooltip("This is an another example")]

可以给属性添加一段描述文字

0.3 Space

[Space(int x)]

添加空格

0.4 TextArea

[TextArea]

将单行文本框变为多行文本框

0.5 SerializeField

[SerializeField]

可以将变量序列化，序列化的意思是说再次读取Unity时序列化的属性是有值的，不需要你再次去赋值，因为它已经被保存下来。

凡是在Inspector窗口中显示的属性都具有Serialize功能，所以public属性默认具有Serialize功能。对于不在窗口显示的private和protected属性，可以在定义属性前添加上述代码，使其在Inspector窗口显示，具有序列化功能。

1 输入Input

1.1 GetAxis()

根据按下按键返回-1~1之间的浮点值。使用这种方法控制人物移动，会有加速度。

movement.x = Input.GetAxis("Horizontal");
movement.y = Input.GetAxis("Vertical");

1.2 GetAxisRaw()

根据按下按键返回-1, 0, 1三者之一的整数值，正负代表方向。没有摩擦力时人物会平滑移动。

movement.x = Input.GetAxisRaw("Horizontal");
movement.y = Input.GetAxisRaw("Vertical");

1.3 GetKey()

当通过名称指定的按键被用户按住时返回true。按键可以通过KeyCode来获取，以下GetKey系列同理。

1.4 GetKeyDown()

当通过名称指定的按键被用户按下的那一帧返回true。

1.5 GetKeyUp()

当通过名称指定的按键被用户松开的那一帧返回true。

2 GameObject

2.1 Instantiate()

详细戳我

该函数是unity中进行实例化的函数，可以将original对象的所有子物体和子组件完全复制，成为一个新的对象，这个对象拥有和源对象完全一样的东西，包括坐标值等。总共有5个重载。

  • original：源对象，用来复制操作的对象物体
  • position：实例化的对象的位置坐标
  • rotation：实例化的对象的旋转坐标（旋转四元数）
  • parent：实例化对象的父对象，在完成实例化函数处理后，实例化对象将在父对象下，是父对象的子对象
  • instantiateWorldSpace：布尔变量，这个值为TRUE，表示实例化对象相对于世界坐标系的位置（是位置，不是坐标值，比如实例化前在Canvas左上角，实例化后还在左上角）不变，相对于父对象的坐标值变了。为false，表示实例化对象相对于父对象的坐标值不变，但在世界坐标系中的位置变了

2.2 Find()

通过名字或路径查找游戏对象

GameObject.Find("GameObject");
GameObject.Find("GameObject/ChildGameObject);

使用规范：

  1. 无法查找隐藏对象，隐藏对象包括查找路径的任何一个父节点隐藏(active=false)。

  2. 不可以跨场景查找。

  3. 如果查找不在最上层，建议合理使用路径查找，路径查找是把双刃剑。

优点：

  1. 解决查找中可能出现的重名问题。
  2. 如果有完全的路径，减少查找范围，减少查找时间。

缺点：

  1. 路径或结构调整后，容易影响到程序中的查找，需要重新定位查找路径。
  2. 如果路径查找中的任何一个父节点active=false，这个对象都将查找不到。
  3. 使用方便但效率低下，此查找相当于递归遍历查找，虽使用方便但效率堪忧，建议在Start()函数中查找对象并保存引用，切忌在Update()中动态查找。

2.3 FindGameObjectWithTag

通过tag来查找gameobject，返回该场景中第一个匹配的对象。

2.4 FindGameObjectsWithTag

通过tag来查找gameobject，返回一个数组，包含所有目标tag的gameobject。如果没有匹配的对象，则返回空。tag必须是存在的。

3 场景SceneManager

需要引入SceneManagement包：

using UnityEngine.SceneManagement;

3.1 LoadScene()

该函数可以实现在多个场景之间切换，以加载上一个场景为例：

SceneManager.LoadScene(SceneManager.GetActiveScene().buildIndex - 1);

GerActiveScene().buildIndex的意思是获取当前场景的编号，场景的前后顺序可以再Building Setting中更改，数值减一是加载上一个场景，如果要加载上一个场景的话就是加1了。

4 时间Time

4.1 Time.time

获取当前时间，以秒为单位，游戏启动时开始计时。

4.2 Time.deltetime

一个极小的时间间隔，返回的是上一帧到这一帧两帧之间的时间间隔。通常使用在Update中，如果要在FixUpdate中使用的话，请换成fixedDalteTime。
